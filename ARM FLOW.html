Key Aspects about the Project

 

●    Our objective will be to create a web portal for connecting Patients, Doctors and Laboratories.

●    We can create a 3 separate portals for Patients, Doctors and Laboratories. They will be connected with each other from database.

●    Our main target will be Patients portal . Except for Patients portal none of them will have idea of existence of others portal.

●    When a patient gets landed on the home page it should looks promising and trustworthy. They should be get know how many doctors and laboratories are connected with us.

●    In home page their will be two options i.e for diagnosis and Laboratory test.

●    In Diagnosis section their will be a direct guide for user to fill his name, age, location, medical problem,etc to filter doctors

●    In laboratory section the user will select details of test he want to done.

●    If user proceed to book any appointment then the user gets redirect to register or login page.

●    After login the portal will show overview of all his selected options or else ask for extra data.

●    If user proceed for confirmation then he/she will redirects to payment page and after payment all confirmation will be displayed. The user will get all details on mobile no. and email given on registration.

●    If user directly register or login he/she will get able to manage his appointment or download appointment details as pdf in dashboard page.

●    On home page the popup first ask the user for location access. After that the user gets recommendation and filters according to its location and ratings of Doctors and laboratories.

●    For Doctors and Laboratories portal first they have to register themselves .

●    After login they will able  to see and manage patients appointments and details.

●    They Laboratories will also gets option to add manage the tests they have with them.

My Approach to Backend

 

●    Libraries Required

○    Express - The Node.Js web framework to handle multiple different http requests and handle API calls.

○    Bcrypt - Used for hashing passwords.

○    JWT - Used for authentication of genuine user.

○    Mongoose:- It is an Object Data Modeling(ODM) library for Mongodb and node.js . It manages relationships between data and also provides schema validation.

○    Password-validator :- It validates the password as rules set by developer by using its various properties and methods.

○    Cors - It allows you to make requests from one website to another website in the browser, which is normally prohibited by another browser policy called the Same-Origin Policy (SOP)

○    Cookie-parser - For storing the user’s sessions.

○    Nodemailer- For sending otp to email.

○    Pdfkit - It is a PDF document generation library for Node and the browser.

○    Dotenv - It loads environment variables from .env file into process.env. It helps to during deployment of project and upload in github.

Approach to Database:-

1) Users:- username, email, name, phone, gender, dob, address, password, cpassword, date, token.

2) Doctors:- Registration_no. , speciality, degree_name ,clinic/hospital_name(if any),years of experience, username, email, name, phone, gender, clinic/hospital_address(if any), password , cpassword, date , token.

3) Laboratories:- Laboratory_name, owner/assistant_name, Laboratory_address, username, email, phone, password, cpassword, date, token.

4) Lab_tests:- Laboratory_name, tests_name, test_cost.

5) Appointment with doctor:- Doctors_name, Date, Time, mode_of_meeting, disease/problem/issue.

6) Appointment with Labs:- Lab_name, test_name,date, time.
